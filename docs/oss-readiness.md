# OSS 공개 체크리스트

이 문서는 릴리스 전에 한 번씩 확인할 항목을 모아둔 체크리스트입니다.

## 배포/릴리스

- [ ] `./gradlew test`가 로컬과 CI에서 통과한다.
- [ ] 릴리스 태그 `v*`에 대해 publish 워크플로우가 동작한다.
- [ ] Maven Central POM 메타데이터가 채워져 있다.
- [ ] Gradle Module Metadata가 유지된다.
- [ ] nmcp 기반 Central Portal 업로드 경로가 문서화되어 있다.
- [ ] 서명 키와 Central Portal 자격 증명 주입 방법이 문서화되어 있다.

## 보안/운영

- [ ] `auth.jwt.secret`과 refresh token 관련 설정의 운영 주입 방법이 문서화되어 있다.
- [ ] 기본 `RefreshTokenStore`가 개발용인지 운영용인지 명시되어 있다.
- [ ] 인증 실패 시 내부 오류가 로그에 남는다.
- [ ] 토큰과 쿠키의 보안 속성(HTTPS, HttpOnly, SameSite)을 문서로 안내한다.

## 사용성

- [ ] 빠른 시작 예제가 복사 가능한 수준으로 제공된다.
- [ ] SPI 구현 가이드가 있다.
- [ ] 샘플 애플리케이션 또는 최소 예제가 있다.
- [ ] 자주 쓰는 설정과 기본 동작이 한 곳에 정리되어 있다.

## 유지보수

- [ ] 기여 가이드가 있다.
- [ ] 보안 이슈 제보 경로가 있다.
- [ ] 릴리스 노트 또는 변경 요약이 있다.
- [ ] 현재 저장소 구조와 실제 코드가 문서에 반영되어 있다.
